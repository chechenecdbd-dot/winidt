# WindInit (Main app for GTA5RP players capt/biz)

Скрытый триггер-бот на C++, реагирующий на изменение цвета прицела в центре экрана.  
Использует **Desktop Duplication API (DXGI)** для захвата области 3×3 пикселя с GPU – нагрузка на CPU практически нулевая.  
Поддерживает циклическое сохранение двух цветов прицела одной кнопкой, переназначение клавиши-триггера, автоматическое ожидание запуска игры и полную очистку цифровых следов.  
Работает без окон, консоли и лог-файлов. **Требуется запуск от имени администратора.**

## ✨ Особенности

- 🎯 **Мгновенный захват области 3×3 пикселя** – через DXGI Duplication, без полных скриншотов. Цвет усредняется по 9 пикселям, что стабилизирует детект при любых режимах сглаживания (TAA, DLSS, FSR).
- 🔁 **Циклическое сохранение цветов** – одно нажатие (`F1`) запоминает нейтральный цвет прицела, следующее – цвет врага (наведи на кого нибудь прицел стоя в ПКМ), и так по кругу.
- ⌨️ **Переназначаемый триггер** – можно выбрать любую клавишу/кнопку мыши для активации, по умолчанию **правая кнопка мыши**.
- 👻 **Полная скрытность**
  - Нет окон, консоли, лог-файлов.
  - Ожидание процесса игры `ecef_process.exe` с полупрозрачным окном, которое не отображается в панели задач.
  - Проверка прав администратора при запуске (без них программа показывает предупреждение и закрывается).
  - Автоматическая очистка `%TEMP%`, `Prefetch` и расширенная зачистка реестра (включая `AppCompatFlags`, `MuiCache`, `ComDlg32`, **BAM**, **UserAssist**).
  - Опциональная маскировка процесса (подмена имени, запрет чтения памяти, самоудаление с диска после перезагрузки).
- 🔊 **Звуковые подтверждения** через звуковую карту (громкость настраивается в коде).
- 🪟 **Совместимость** – Windows 10/11 (все сборки, 1507–25H2 и новее), включая полноэкранный режим игр.

## ⌨️ Горячие клавиши

| Клавиша          | Действие |
|------------------|----------|
| `Delete`         | Включить / выключить бота (звуковой сигнал: 1000 Гц – вкл, 500 Гц – выкл) |
| `F1`             | Сохранить цвет прицела (первое нажатие – нейтральный, второе – вражеский) |
| `F2`             | Переназначить клавишу-триггер (нажать, затем нужную кнопку/клавишу) |
| `Ctrl+Shift+Q`   | Завершить программу (перед закрытием выполняется финальная очистка следов) |

Триггер по умолчанию: **правая кнопка мыши (ПКМ)**.  
Звуковые сигналы подтверждают все операции.

## 📦 Как использовать

1. **Запустите `windinit.exe` от имени админа.**  
   Если неправильно, появится предупреждение на 5 секунд и программа закроется.

2. **Ожидание игры.**  
   После успешного запуска ты увидишь полупрозрачное окно с надписью «Wait run GTA5/RAGEMP». Оно будет оставаться на экране до тех пор, пока не запустится процесс `ecef_process.exe`. Когда игра стартует, окно исчезнет автоматически.

3. **Запомните цвета прицела:**
   - Наведите прицел на нейтральную область (стена, небо), где цвет не меняется.
   - Нажмите `F1` — услышите низкий звук (800 Гц). Сохранён нейтральный цвет.
   - Наведите прицел на врага (цвет должен измениться).
   - Снова нажмите `F1` — услышите высокий звук (1200 Гц). Сохранён вражеский цвет.
   - Повторные нажатия `F1` циклически перезаписывают нейтральный → вражеский → нейтральный…

4. **Активируйте бота:** нажмите `Delete` (звуковой сигнал включения: 1000 Гц – вкл, 500 Гц – выкл).

5. **В игре зажмите настроенную клавишу-триггер** (по умолчанию ПКМ).  
   Когда усреднённый цвет области 3×3 пикселей в центре экрана совпадёт с сохранённым вражеским цветом, бот автоматически нажмёт левую кнопку мыши (ЛКМ) с интервалом около 10 мс.

6. **Переназначение триггера (опционально):**
   - Нажмите `F2` (два коротких сигнала).
   - Нажмите любую клавишу клавиатуры или кнопку мыши, которую хотите использовать как триггер.
   - Успешное назначение подтверждается высоким сигналом (1500 Гц). Если в течение 5 секунд ничего не нажато – низкий сигнал ошибки.

7. **Выход:** `Ctrl+Shift+Q` – программа выполнит финальную очистку всех следов и закроется.

## ⚙️ Принцип работы

1. При старте проверяются права администратора, затем инициализируется устройство Direct3D 11 и дупликатор рабочего стола (DXGI 1.2).  
2. Однократно создаётся переиспользуемая staging-текстура размером 3×3 пикселя.  
3. Основной цикл каждые ~5 мс копирует с GPU область 3×3 вокруг центра экрана и вычисляет средний цвет.  
4. При включённом боте и зажатом триггере средний цвет сравнивается с вражеским (допуск ±30 по сумме каналов RGB). При совпадении эмулируется левый клик.  
5. Горячие клавиши обрабатываются в отдельном потоке.

## 🧹 Очистка следов

Очистка выполняется **дважды** – при запуске (сразу после появления процесса `ecef_process.exe`) и при выходе (`Ctrl+Shift+Q`). Это исключает накопление следов во время работы.

Автоматически удаляются:

- Временные файлы в `%TEMP%`, соответствующие маске `windinit*.exe`.
- Prefetch-файлы из `C:\Windows\Prefetch`, начинающиеся с `windinit.exe`.
- Записи реестра (без пропусков, с корректным перебором даже после удаления):
  - `HKCU\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Layers`
  - `HKCU\Software\Classes\Local Settings\Software\Microsoft\Windows\Shell\MuiCache`
  - `HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\ComDlg32\OpenSavePidlMRU`
  - `HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\ComDlg32\LastVisitedPidlMRU`
  - **BAM** (Background Activity Moderator) – удаляются все значения, содержащие `windinit.exe`, из `HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings\{SID}`.
  - **UserAssist** – расшифровываются (ROT13) имена значений в разделе `HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist\{GUID}\Count`; при совпадении пути с `windinit.exe` запись удаляется.

Дополнительная маскировка:
- Подмена имени процесса в PEB.
- Установка пустого DACL, запрещающего другим процессам открывать нашу программу.
- Отложенное удаление собственного исполняемого файла с диска при следующей перезагрузке.
